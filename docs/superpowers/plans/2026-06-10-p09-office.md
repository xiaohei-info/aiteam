# P09 Office Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改北向 API 的前提下，为 P09 办公室动态页补齐更接近 PRD 的场景浏览交互，优先完成拖拽平移与滚轮缩放，并保持既有办公室合同与页面能力不回退。

**Architecture:** 保持 Team Panel `office scene/feed` 聚合合同不变，只在 `office.js` 增强视口交互层。测试先行，先让 layer4 前端测试证明缺口存在，再以最小实现补齐行为。

**Tech Stack:** Python pytest, Node VM-based frontend module tests, vanilla JavaScript, existing Team Panel office APIs

### Task 1: 为办公室页补拖拽/滚轮交互测试

**Files:**
- Modify: `app/tests/aiteam/layer4_frontend_bff/test_office_page.py`
- Test: `app/tests/aiteam/layer4_frontend_bff/test_office_page.py`

- [ ] **Step 1: 写失败测试**

新增一个 Node VM lifecycle，模拟：

- `wheel` 事件把 `scale(1)` 变成更大的缩放
- `mousedown -> mousemove -> mouseup` 把 `translate(0px, 0px)` 变成非零位移

- [ ] **Step 2: 跑测试确认失败**

Run: `pytest app/tests/aiteam/layer4_frontend_bff/test_office_page.py -q`

Expected: 新增测试 FAIL，原因是当前 `office.js` 尚未绑定 wheel / drag 行为

- [ ] **Step 3: 写最小实现**

在 `app/static/aiteam/pages/office.js` 中给办公室视口绑定：

- `wheel` -> `zoomViewport`
- `mousedown/mousemove/mouseup/mouseleave` -> `panViewport`

要求：

- 保留原有按钮式放大/缩小/平移/重置
- 不破坏 `transform` 现有格式
- 不依赖浏览器专属 API 以外的新基础设施

- [ ] **Step 4: 跑测试确认通过**

Run: `pytest app/tests/aiteam/layer4_frontend_bff/test_office_page.py -q`

Expected: PASS
