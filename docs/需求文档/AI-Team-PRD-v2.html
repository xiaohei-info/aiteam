P01 登录页 企业前台 · PC客户端 · 入口页
🖥️ PC 客户端
P01 · 登录页原型图
交互逻辑
  • 默认展示微信扫码Tab，加载时自动请求二维码
  • 点击「手机号登录」Tab：切换为手机号+验证码表单
  • 扫码后App出现「确认登录」按钮，客户端轮询（3s/次）状态
  • 二维码到期（180s）后自动灰化并显示「点击刷新」按钮
  • 手机号发送验证码：60s冷却，提交后校验有效期（5min）
  • 登录成功→动效淡出→进入主界面（新用户→引导招募首个员工）
  • 记住登录状态（token刷新机制），下次启动自动登录
数据逻辑
  • 微信登录：接入微信开放平台OAuth2，获取 unionid
  • 手机号登录：短信验证码，关联用户表 phone 字段
  • 首次登录自动创建企业账号（orgId），绑定owner角色
  • JWT token：access_token（2h）+ refresh_token（30d）
  • Token 存储在本地 Keychain（加密）
  • 并发登录：同一账号最多3个设备，超出踢掉最早设备

字段规范

字段类型必填说明
手机号string(11)必填国内11位，正则校验 /^1[3-9]\d{9}$/
验证码string(6)必填6位数字，5min内有效
微信codestring系统扫码回调，服务端换取 unionid/openid
device_idstring系统客户端硬件指纹，随请求自动携带
remember_mebool可选默认 true，控制 refresh_token 有效期
⚠ 手机号登录需做频率限制：同一手机号 60s 内只能发送1次验证码；同一IP每分钟最多10次请求。
P02 主界面·私聊入口 企业前台 · PC客户端 · 核心主页
🖥️ PC 客户端
P02 · 主界面原型图（新用户空态）
交互逻辑
  • 新用户空态：左侧列表显示虚线占位区+「前往人才市场」按钮，右侧显示引导文案
  • 有员工后：左侧列表展示员工头像、名称、最后一条消息摘要、未读数角标
  • 左侧图标栏：私聊/群聊/组织/知识库/人才市场/设置，Tooltip 悬停提示
  • 点击员工→右侧展示对话区（P05），高亮选中态
  • 员工列表支持搜索（顶部搜索框），实时过滤名称/角色
  • 右键员工项→快捷菜单：查看详情/设置为星标/解雇
数据逻辑
  • 接口：GET /api/org/agents 获取当前企业已招募员工列表
  • 员工列表按「最近交互时间」倒序排列，星标置顶
  • 未读消息数：本地存储已读游标（last_read_msg_id），服务端返回 unread_count
  • 员工状态字段：online/busy/idle/offline，头像右下角小圆点显示
  • 首次加载：骨架屏 → 数据渲染（≤500ms），离线缓存7天

P03 人才市场 企业前台 · PC客户端 · 发现与招募
🖥️ PC 客户端
P03 · 人才市场原型图
交互逻辑
  • 顶部搜索栏：输入实时触发模糊搜索（debounce 300ms），搜索范围：专家名称、标签、技能名
  • 分类 Tab：单选，选中后过滤专家卡片（前端过滤或后端搜索视数量而定）
  • 每张卡片展示：头像、岗位名、底层大模型、Skills数量、招募企业数
  • 「招募」按钮：点击弹出确认弹窗，确认后触发招募动画（卡片飞入侧边栏）
  • 已招募的专家：按钮变为「已招募」灰色禁用态，卡片右上角显示绿色✓
  • 点击卡片主体区域（非按钮）→ 跳转专家详情页（P04）
  • 无限滚动加载（或分页），每次加载20条
数据逻辑
  • 接口：GET /api/marketplace/agents?category=&keyword=&page=
  • 排序：默认按「热度」（招募数）降序，可切换「最新上架」
  • 招募接口：POST /api/org/recruit，body: { agent_template_id }
  • 招募后服务端复制模板创建实例agent_id，初始化记忆/知识库关联
  • 每个企业同一岗位类型限制招募3人（可配置），超出提示升级套餐

P04 专家详情页 企业前台 · PC客户端 · 招募前查看
🖥️ PC 客户端
P04 · 专家详情页原型图（抽屉/侧边面板）
交互逻辑
  • 以右侧抽屉形式从右滑入（动效：translateX 420→0，300ms ease），背景蒙层
  • 4个Tab切换：基本信息 / 技能列表 / 知识库 / 初始记忆
  • 「立即招募」按钮：点击后显示加载状态（旋转icon），成功后触发招募成功动画
  • 招募成功动画：专家头像从详情页飞出→缩放进入左侧员工列表，同时播放粒子爆炸效果
  • 点击蒙层或 ✕ 关闭抽屉（反向动画）
数据逻辑
  • 接口：GET /api/marketplace/agents/:id 获取专家完整信息
  • 技能列表：展示 skill_name / skill_description / skill_source（clawhub/skillhub）
  • 知识库：展示模板关联的知识库文件列表（企业招募后可替换）
  • 初始记忆：展示 system prompt 中预置的 memory 条目
  • 招募：复制模板数据，创建专属实例，关联当前 org_id
P05 单聊对话页 企业前台 · PC客户端 · 核心使用场景
🖥️ PC 客户端
P05 · 单聊对话页原型图（三栏布局）
交互逻辑
  • 三栏布局：图标栏(48px) + 员工列表(220px) + 对话区(flex) + 配置面板(122px，可折叠)
  • 消息发送：Enter发送 / Shift+Enter换行 / Ctrl+Enter发送（可配置）
  • AI回复：流式输出（SSE），逐字打印动效（cursor闪烁），完成后消失
  • 消息操作：悬停气泡 → 出现工具栏（复制/引用/点赞/重新生成/删除）
  • 支持 @提及：输入 @ 弹出员工选择菜单，群聊中可 @特定员工
  • 消息类型：文本/图片/文件（拖拽上传）/Markdown代码块/工具调用卡片
  • 右侧面板「配置员工」：跳转至后台 B01 页面进行深度配置
数据逻辑
  • 消息协议：SSE（Server-Sent Events）实时流式输出
  • 消息存储：SQLite本地 + 服务端同步，离线可查历史
  • 上下文窗口：默认最近20轮，超出时自动截断（保留系统提示词）
  • 工具调用显示：tool_call 消息显示为「卡片」，展示工具名称、参数、执行结果
  • 文件上传：图片(≤20MB)、PDF/Word(≤50MB)，OSS存储，生成临时访问URL
  • Token消耗：每次对话完成后异步更新 agent_token_usage 表

P06 群聊页面 企业前台 · PC客户端 · 多员工协作
🖥️ PC 客户端
群聊允许用户同时与多个数字员工对话，员工之间可相互协作、接续回复，实现多角色业务讨论。
交互逻辑
  • 「新建群聊」按钮：弹窗选择多个员工（最多10人），填写群名称
  • 发送消息后，系统根据消息内容自动路由给最匹配的员工回复
  • @提及指定员工：强制触发该员工优先回复
  • 每条AI回复前显示员工头像+名称气泡区分发言者
  • 员工可「接力」：一个员工回复后，可触发另一员工补充
  • 群聊头像：取前4名员工头像拼成2×2宫格
  • 支持解散群聊（不删除历史记录）、踢出员工、新增员工
数据逻辑
  • 群聊表：group_chat（id/name/org_id/agent_ids[]）
  • 路由逻辑：消息发出后服务端调用 Router Agent 分析意图，返回最匹配的 agent_id
  • 同一消息最多触发3个员工回复（防止死循环）
  • 群聊消息与单聊消息共用 messages 表，通过 chat_type 区分
  • 成员变更（加入/退出）记录到 group_member_log 表

群聊数据字段

字段类型说明
group_idstring(uuid)群聊ID
group_namestring(50)群聊名称
org_idstring所属企业
membersjson[]成员列表：[{agent_id, join_time}]
owner_user_idstring创建人用户ID
created_attimestamp创建时间
P07 组织架构 企业前台 · PC客户端 · 人员结构管理
🖥️ PC 客户端
P07 · 组织架构页原型图
交互逻辑
  • 树形图展示，可缩放（鼠标滚轮）、平移（拖拽空白区域）
  • 点击员工节点：右侧弹出员工详情面板（查看/配置）

P08 知识库 企业前台 · PC客户端 · 企业级知识管理
🖥️ PC 客户端
交互逻辑
  • 知识库列表：显示公司级知识库 + 各部门/员工级知识库
  • 创建知识库：填写名称/描述，选择权限（全员可见/部门可见/仅指定员工）
  • 上传文档：支持拖拽、批量上传，格式：PDF/Word/TXT/Markdown/Excel/CSV
  • 第三方接入：点击「连接外部文档」→ 弹窗选择腾讯文档/钉钉文档/Notion等
  • 文档处理状态：上传→解析中（进度条）→向量化→就绪
  • 知识库与员工关联：在员工配置页选择使用哪个知识库
  • 搜索：在知识库内搜索文档内容（语义搜索，非关键词）
数据逻辑
  • 文档上传→服务端进行文本切块（chunk_size: 512 tokens）
  • 向量化：调用 embedding API，存储到向量数据库（Qdrant）
  • RAG检索：每次对话时，自动从关联知识库检索 topK=5 相关片段
  • 第三方同步：OAuth授权后，定时（每小时）拉取文档更新
  • 权限字段：visibility（public/department/private）+ allowed_agent_ids

P09 办公室动态 企业前台 · PC客户端 · 员工工作状态可视化
🖥️ PC 客户端
P09 · 办公室页面原型图（Marvis风格等距立体图）
交互逻辑
  • 等距立体（Isometric）办公室视图，每个员工对应一个工位
  • 工位节点有动效：工作中→键盘打字动画；繁忙→思考气泡飘动；就绪→呼吸光效
  • 悬停工位→浮现当前任务气泡（显示正在执行的任务名称）
  • 点击工位→右侧弹出员工详情（当前任务/最近对话/历史任务列表）
  • 支持全屏模式（快捷键 F）
  • 实时刷新（WebSocket推送状态变更）
动效规范
  • 工作中：工位屏幕闪烁 + 顶部思考气泡（浮动 loop 动画，周期3s）
  • 就绪：工位节点呼吸灯（绿色脉冲，周期4s，scale 1→1.05）
  • 状态切换：工位节点颜色渐变过渡（duration 600ms）
  • 新任务开始：任务气泡从下方弹出（translateY 20→0，opacity 0→1，200ms）
  • 离线：工位灰化（opacity 0.4），无动效
⚙️二、企业后台（PC 客户端）
管理员/Owner专属后台，用于深度配置数字员工、管理成本、接入外部系统。与前台共用同一客户端，通过顶部「切换后台」按钮进入。
B01 数字员工管理 企业后台 · PC客户端 · 核心管理页
⚙️ 企业后台
B01 · 数字员工管理页原型图
交互逻辑（配置弹窗）
  • 点击「配置」→弹出配置面板（右侧抽屉，宽560px）
  • 配置Tab 1 - 基础信息：名称、头像上传、岗位描述（Prompt编辑器，支持高亮）
  • 配置Tab 2 - 大模型：下拉选择模型（GPT-4o/Claude/DeepSeek/Qwen等），Temperature滑块，Max Tokens输入
  • 配置Tab 3 - 技能：已启用技能列表（开关切换），从技能市场添加技能
  • 配置Tab 4 - 知识库：关联/取消关联知识库
  • 配置Tab 5 - 记忆：查看/编辑/删除记忆条目
  • 「解雇」：二次确认弹窗→软删除（保留对话记录，标记为 archived）
数据逻辑
  • 接口：GET /api/admin/agents 获取企业所有员工
  • 更新：PATCH /api/admin/agents/:id 更新配置
  • 解雇：DELETE /api/admin/agents/:id（软删除，status='archived'）
  • 本月消耗：从 agent_token_usage 表按月聚合计算
  • 配置变更即时生效（下次对话时采用新配置）

B02 技能市场 企业后台 · PC客户端 · 对接ClawHub/SkillHub
⚙️ 企业后台
交互逻辑
  • 页面分两区：已安装技能（当前企业）+ 市场浏览（ClawHub/SkillHub）
  • 技能卡片：技能图标、名称、来源（clawhub.io标注）、描述、安装企业数
  • 点击「安装」→技能添加到企业技能库，可在员工配置中启用
  • 技能权限：全企业共享 / 仅指定员工可用（安装时配置）
  • 已安装技能：可查看版本、更新、卸载
  • 搜索：关键词搜索技能名称、描述、标签
数据逻辑
  • 市场数据：调用 ClawHub API 拉取技能列表（定时缓存，每10min刷新）
  • 安装：本地创建 org_skill 关联记录，下载技能配置文件
  • 技能执行：通过 MCP 协议调用，server端维护技能进程池
