产品概述
Product Overview

AI Team 是一个面向企业和个人的 智能体团队协作平台。它将多个具备独立角色、专项能力的 AI 智能体组织在统一的工作空间中，用户可以像管理真实团队一样，创建智能体成员、为其分配岗位与技能、搭建组织架构，并通过私聊或群聊的方式发起协作任务。智能体之间能够自主分工、互相委托、共享上下文，最终合力完成复杂的多步骤任务。

核心定位： 下一代 AI 工作站——把分散的 AI 对话工具，升级为一支可以持续运转、独立决策、协同执行的 AI 员工队伍。
🧩

可组织

像管理真实团队一样管理 AI 成员，支持完整的组织架构体系（部门/岗位/职级）

🤝

可协作

智能体之间自主分工委托，用户只需输入任务目标，剩下的交给团队自动完成

🧠

可进化

每个智能体拥有独立记忆和持续运行能力，越用越懂你的业务场景

🔌

可扩展

技能市场支持按需为智能体装配新能力，一键安装即用，低门槛扩展

🔍

可追溯

完整的系统日志、对话历史和任务上下文，每一步执行都清晰可查

🏠

本地安全

纯本地化部署，数据存储在本地 SQLite，完全符合企业数据隐私要求

需求背景
Background & Problem Statement
😤 核心痛点分析

个人用户痛点

  • 独立工作者需要在写作、分析、编程、设计之间频繁切换工具
  • 反复粘贴上下文，效率极低，一遍遍重复同样的任务说明
  • 单一 AI 工具只能完成单点任务，无法串联完整工作流
  • 多个 SaaS 工具分散，管理成本高，学习曲线陡峭

企业/团队痛点

  • 人手不足，重复性知识工作（写报告、分析数据）耗时巨大
  • 单点 AI SaaS 工具碎片化，管理成本与授权费用叠加
  • 无法按照真实组织架构管理 AI 能力，难以系统化落地
  • AI 能力无法持续积累沉淀，每次使用都从零开始
💡 产品机会

AI 从"工具"走向"员工"的历史窗口期——现有平台要么是无协作的单智能体对话（ChatGPT 等），要么是对技术用户要求极高的工作流编排平台（n8n/Dify）。AI Team 填补了两者之间的空白：低门槛的多智能体协作操作系统。

📊 市场定位
  • 个人 私人多角色 AI 工作团队
  • 企业 AI 员工扩充方案
  • 开发者 开放的多智能体实验平台

功能模块详细说明
Feature Specifications
🤖
5.1 智能体管理（Agent Management）
智能体是 AI Team 的核心构成单元，代表平台中的每一位 AI "员工"
P0 核心功能

创建与配置

  • 自定义名称、岗位名称、头像表情符号
  • 三大主流模型系列选择（DeepSeek / GLM / Doubao）
  • 任务类型自由切换底层模型
  • 系统提示词（System Prompt）深度定制

核心能力

  • 每个智能体可深度定制为垂直领域专家
  • 同平台内多个专业智能体并存
  • 按业务需求分配不同模型，控制成本
  • 智能体归属部门/岗位分配
💡 核心价值

不同于通用 AI 助手，每个智能体都可以被深度定制为某个垂直领域的专家，在特定任务上显著优于通用对话。同一平台内多个专业智能体并存，覆盖不同职能需求，彻底替代了多个单点工具的组合使用。

💬
5.2 私聊对话（Private Chat）
每个智能体拥有独立的私聊频道，适合专注于单一专业领域的任务场景
P0 核心功能
  • 多媒体内容支持（图片最大 20MB + 文件）
  • 图像生成意图自动识别，调用 CogView API
  • 推理链（Chain of Thought）折叠展示面板
  • 历史记录持久化至本地 SQLite 数据库
  • 支持完整历史回溯，频道可删除重开
  • 数据完全在用户本地，不存在隐私泄露风险
👥
5.3 群聊协作（Group Chat）
AI Team 最具差异化竞争力的核心功能
P0 核心功能
  • @提及方式同时激活多个智能体
  • 多智能体并发响应（不同视角独立回复）
  • 适合需要综合评估、多方论证的决策场景
  • 灵活成员配置（随时添加/移除智能体）
  • 设置群组名称和目的说明
  • 群组持久化，作为真正的"项目工作间"
🦞
5.4 龙虾编排协议（Lobster Orchestration）
AI Team 最核心的技术创新，真正实现"智能体自主协作"的关键引擎
技术核心

工作原理

  • 智能体通过系统提示词自动具备协作能力
  • 回复末尾嵌入 <delegate> XML 指令块
  • 声明子任务委托对象及依赖关系
  • 按依赖关系自动排序执行子任务

用户体验

  • SSE 实时推送执行进度至前端
  • 协同气泡动态展示每个智能体执行结果
  • 可折叠详情卡，界面整洁信息不丢失
  • 主持智能体自动生成汇总回复
💡 核心价值

龙虾编排将"用户给指令、AI 给回答"的单轮交互，升级为"用户定目标、AI 团队自主完成"的多步骤工作流。这是从 AI 工具到 AI 员工的本质跃升。

🔄
5.5 OpenClaw Agent Loop & 记忆系统
让智能体在没有用户实时干预的情况下自主工作
差异化能力

PDAR 循环

  • 感知（Perceive）：扫描对话消息和系统状态
  • 决策（Decide）：结合目标、记忆、工具决定行动
  • 行动（Act）：执行决策，调用技能工具接口
  • 反思（Reflect）：结果存入长期记忆

长期记忆类型

  • fact（事实）
  • experience（经验）
  • reflection（反思）
  • plan（计划）+ tool_execution（工具执行结果）
Loop 生命周期控制
▶ 启动 → ⏸ 暂停 → ▶ 恢复 → ⏹ 停止
执行间隔精确控制（毫秒级）
🛍️
5.6 技能市场（SkillHub / ClawHub）
智能体能力的扩展体系，按需扩展按角色配置
P1 重要功能

内置技能（8项）

🔍 联网搜索 💻 代码执行 🎨 图像生成 📁 文件处理 📊 数据分析 📧 邮件发送 📅 日历管理 📚 知识库问答

ClawHub 市场

  • 按分类浏览 + 关键词搜索
  • 多维排序（最新/最热/最高评分）
  • 一键安装/卸载
  • SKILL.md 低门槛扩展机制
🏢
5.7 组织架构（Org Chart）
面向企业场景的核心功能模块，将"一堆智能体"升级为"结构化的 AI 组织"
P1 重要功能
  • 多层级部门体系（支持嵌套子部门）
  • 岗位管理（1~5 级职级 + 颜色标识）
  • 智能体归属部门/岗位分配
  • 统计看板：部门总数/岗位总数/分配比例
  • 未分配智能体统一展示区域
  • 为权限管理、报表等企业功能奠定数据基础

⚡ 数据流向
  • 前端发送消息 → Express API → SQLite 写入
  • AI API 调用 → SSE 流式推送 → 前端实时渲染
  • Loop 执行结果 → 记忆系统评分 → SQLite 持久化
  • 技能调用 → ToolRegistry → 工具执行结果回写
  • <delegate> 解析 → 任务队列 → 子任务依次调度

-- agents 表
CREATE TABLE agents (
  id          INTEGER PRIMARY KEY,
  name        TEXT NOT NULL,
  role        TEXT,                -- 岗位名称
  avatar      TEXT,                -- emoji 表情
  model       TEXT NOT NULL,        -- deepseek-chat / glm-4-flash 等

  loop_interval INTEGER DEFAULT 60000, -- 毫秒
  loop_goal     TEXT,

CREATE TABLE messages (
  id              INTEGER PRIMARY KEY,
  conversation_id INTEGER NOT NULL,
  role            TEXT NOT NULL,     -- user / assistant / system

  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP

技术架构亮点
Technical Architecture Highlights
🏠 纯本地化部署

整个系统基于 Node.js + Express + SQLite 构建，无需云数据库，所有数据存储在本地 data.db 文件中。完全符合企业数据隐私要求，适合内网环境部署，数据不出境。

⚡ 实时通信（SSE）

多智能体编排任务通过 Server-Sent Events 协议实时推送进度，用户看到的每一步执行结果都是真实的实时流，而非轮询模拟。连接断开后支持自动重连。

🛡️ 请求健壮性

前端所有 API 请求统一封装 10 秒超时机制（AbortController），即便服务器暂时不可达，界面也不会卡死，而是给出明确的超时提示，提升系统容错能力。

🔄 数据库迁移机制

系统内置增量 Schema Migration 逻辑，升级版本时自动为旧数据库补齐新增字段，保证平滑升级不丢数据，无需用户手动操作或备份。
